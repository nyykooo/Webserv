HTTP METHODS

HTTP defines special words called methods to describe what the client wants:

GET:
The client sends a request line like:

GET /path/to/file.txt HTTP/1.1
Host: hostname
Content-Length: 32

This is the body of the request!

And the server search for the file requested, read it's content and send it to the client.

You can test an GET request to any page you want by typing the name of the file on the textarea bellow and clicking the button GET

POST: Used to send data to the server.
Example: submitting a form, uploading a file, creating a new record in a database.
Not idempotent (repeating may cause duplicates).

You can test an POST request by selecting the file when clicking the select select file button bellow and clicking the button POST

DELETE: Used to remove data on the server.
Example: deleting a user account or removing a file.
Idempotent (deleting the same resource twice still results in "gone").
